# vauban

Claude Code・Gemini・Qodo Merge と静的解析ツールを組み合わせたマルチレイヤーコードレビュー構成の設計メモ。

コミット → プッシュ → CI → PR の各段で多重に防壁を張り、最後の砦として人間の最終確認（**Zion check**）を置く。

実装は `setup.sh` として完成している。

→ [DESIGN.md](DESIGN.md)

## なぜ Vauban か

セバスティアン・ル・プレストル・ド・ヴォーバン（1633–1707）は、ルイ14世に仕えたフランスの軍事技師。星形要塞と稜堡システムによって**「縦深防御（defense in depth）」を体系化した人物**で、本構成の思想そのものを体現している。

- **数に縛られない多層構造**：三つ首のケルベロスのように「3」を背負う名前だと、レイヤーを増やすたびに名前と実態がズレる。ヴォーバンの要塞は「外堡 → 稜堡 → 主郭」と防壁を**何段でも重ねられる**。pre-commit → pre-push → CI → PR と段を足していける本構成に合う。
- **力押しでなく手順で守る**：ヴォーバンの攻城・築城は数学的に手順化されており、無駄な突撃＝死傷者を減らすための合理主義だった。自動チェックを段階的に積む発想と重なる。
- **背負う人物像が良い**：主君ルイ14世は専制と浪費のアイコンだが、ヴォーバン本人は叩き上げの技術者で、晩年は貧者の負担を軽くする公平な税制を提言して王の不興を買い失脚した「**権力に物申した良識の人**」。エンジニアリングの名前として担ぐにふさわしい。
- セキュリティ界隈で**ほぼ未使用**。Cerberus / Aegis / Sentinel のような擦られた名前と被らない。

## なぜ Zion check か

最終の人間レビュー層の愛称。サッカー日本代表のGK鈴木彩艶（Zion）にちなむ。

- **GK ＝ 最後の砦（last line of defense）**：すべての防壁をすり抜けたものを最後に止める守護神。多層防御の「最終ライン」そのもの。
- **「Zion（シオン）」の二重の含意**：聖書では城砦の丘・要塞都市、『マトリックス』では人類最後の砦。要塞（Vauban）の最奥に立つ守り手、という構図が名前のレベルで噛み合う。
- **自動化しない層であることの宣言**：複数の AI が「問題なし」と言った後ほど人間の注意は下がる。Zion check は省略・形骸化させない、という戒めを名前に込めている。AI の指摘がない状態でのマージは「問題がないこと」を意味しない。
